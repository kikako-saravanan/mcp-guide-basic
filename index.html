üöÄ 6-Step Workflow

1
User Request
You ask Claude a question that requires external data or actions.
Example: "Show me open GitHub issues in my project"
2
Host Analysis
The Host analyzes your request and determines what external data or tools are needed.
AI identifies the need for GitHub data and the required MCP server
3
Client Communication
MCP Client sends structured request to appropriate MCP Server using JSON-RPC protocol.
Client sends "fetch_github_issues" request with your project parameters
4
Server Execution
MCP Server performs the actual work (API calls, database queries) and processes data.
GitHub Server calls GitHub API, retrieves issues, prepares response
5
Response Integration
Claude incorporates external data into context and generates comprehensive response.
AI analyzes GitHub data and prepares detailed project status
6
User Experience
You receive detailed, current response as if Claude "knew" this information all along.
"You have 12 open issues: 3 high priority, 9 medium priority..."

üí° Real-World Examples

üöÄ Building Offline MCP for Design System

1
Create Simple Script (Not a Web Server!)
Your "MCP Server" is just a Node.js script that reads local NPM packages and outputs JSON via stdio. 
// This is NOT a web server - just a local script!
#!/usr/bin/env node
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import * as BdsTokens from '@lowes-tech/bds-tokens';
import * as BdsReact from '@lowes-tech/bds-react';
2
Read NPM Packages Directly
Access your design system packages that are already installed locally. 
// Read tokens from installed package
const colorTokens = extractTokensFromTheme(BdsTokens.ThemeVariables);
const componentList = Object.keys(BdsReact);
3
Configure Windsurf (Local Process)
Point Windsurf to your local script - no network involved! 
{
  "mcpServers": {
    "backyard": {
      "command": "node",
      "args": ["/path/to/your/backyard-mcp-script.js"]
    }
  }
}
4
Communication via stdio
Windsurf talks to your script through standard input/output - no HTTP! 
Windsurf ‚Üí stdin ‚Üí Your Script ‚Üí stdout ‚Üí Windsurf
(All happens locally on your machine)

üí° Key Insight

MCP "Server" = Local Process, Not Web Server

Your script reads local NPM packages and exposes design tokens/components through the MCP protocol. No networking, no web servers, no infrastructure!
